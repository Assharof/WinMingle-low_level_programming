
## Description
This project focuses on debugging C programs by identifying and fixing errors in existing code.

Debugging is the process of finding and correcting mistakes (bugs) in software that prevent it from running correctly.

## Learning Objectives

At the end of this project, students should be able to:

- Understand what debugging means
- Identify common programming errors
- Interpret compiler error messages
- Correct logical and syntax errors
- Test programs using custom main files

## Requirements

- Allowed editors: vi, vim, emacs
- All files must end with a new line
- Code must compile on Ubuntu 20.04 LTS
- Compilation command:
gcc -Wall -Werror -Wextra -pedantic -std=gnu89
Copy code

- Each directory must contain a README.md file

## Project Tasks

### Task 0 - Multiple mains
Create a main file that tests the function `positive_or_negative()` using `0` as input.

### Task 1 - Infinite loop
Fix an infinite loop by commenting out the problematic line in the program.

### Task 2 - Largest number
Debug the function `largest_number()` to correctly return the largest of three integers.

### Task 3 - Leap year
Fix the function `print_remaining_days()` to correctly handle leap years.
